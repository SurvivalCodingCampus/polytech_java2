# 2026-06-09 클래스

## 오늘 배운 내용

- Stack과 Heap
    - Stack, Heap은 실행 중 메모리할당
        - 자바에서는 new 같은 경우가 Heap, 동적할당 malloc()
        - Stack에는 주로 지역변수, 매개변수, 참조값이 저장
            - Hero hero1 = new Hero(); // Hero 주소로는 접근하기 어려워서, hero1 변수(메모리 번지가 들어있음)로 사용하여 접근
            - hero1.hp = 100;
            - Hero hero2 = Hero1; // new가 한 번만 사용되었기 때문에 Heap에는 Hero 객체가 하나만 생성
            - hero2.hp = 200;
            - System.out.println(hero1.hp); // 결과: 200
        - 참조값은 객체의 위치를 가리키는 값

- int, double, float, bool, char 등 값을 저장하는 기본형(Primitive) 타입
    - 기본 타입은 Default 값이 있음
        - int, short, long = 0, char = \u0000, boolean = false
    - this를 사용하면 기본값이 없으므로 기본값을 넣으려면 기본생성자를 사용하면 됨

- 생성자는 객체를 생성할 때 자동으로 실행되는 메서드
- 생성자의 이름은 클래스 이름과 같아야 함
- 생성자는 반환 타입을 작성하지 않음
- new 로 생성되는 인스턴스인 참조형(Referance) 타입
    - String은 레퍼런스 타입, 초기값: int[] = null, String = null
    - 인스턴스 생성 방법
        - Hero hero1 = new Hero("홍길동", 100);

- 정적 멤버
    - static (정적) 변수
        - 클래스가 메모리에 로딩될 때 생성
        - 사용법) 클래스명.변수명
        - static 필드끼리 공유
            - class 안에 static 이 있어도 접점이 없어서 class의 변수를 끌어서 쓸 수 없음.

- 테스트의 장점
    - 장애에 관한 신속한 피드백
    - 개발 주기에서 조기 장애 감지
    - 회귀에 신경 쓸 필요 없이 코드를 최적화할 수 있도록 하는 더 안전한 코드리팩터링

## 기억할 것

- Stack에는 변수와 참조값이 저장되고, Heap에는 new로 생성된 실제 객체가 저장됨
- 자바 관례상 상수는 대문자로만 표현
- 되도록 매직넘버를 사용하지 말고 변수로 사용
- static 은 class의 변수를 끌어서 사용할 수 없음
- 생성자는 클래스의 이름과 같아야 함
- 초기값이 있는 기본형(Primitive)타입과 없는 참조형(Referance)타입

## 실습 코드

```java
package com.survivalcoding;

public class Cleric {
    static final int MAX_HP = 50;
    static final int MAX_MP = 10;
    static final int SELF_AID_MP_COST = 5;

    String name;
    int hp;
    int mp;

    public Cleric(String name) {
        this(name, MAX_HP, MAX_MP);
    }

    public Cleric(String name, int hp) {
        this(name, hp, MAX_MP);
    }

    public Cleric(String name, int hp, int mp) {
        this.name = name;
        this.hp = hp;
        this.mp = mp;
    }
}


```

## 어려웠던 점

- 생성자는 가능한 한 중복되는 코드가 없도록 줄이는 코드
- Test에서 생성자가 없는 데 검증하는 방법

## 해결 방법

- ChatGPT를 활용하여 문제법 해결
    - this.필드명은 필드와 매개변수를 구분할 때 사용하고, this(...)는 같은 클래스의 다른 생성자를 호출할 때 사용.
        - this(...)는 생성자 안에서 반드시 첫 줄에 작성
    - Cleric.class.getDeclaredConstructor();을 사용해서 매개변수가 없는 생성자를 찾음.
    - 생성자가 없으면 NoSuchMethodException이 발생하여 테스트 성공
