# 2026-06-15 컬렉션

## 오늘 배운 내용

- 대표적인 컬렉션
    - List
        - 순서 대로 쌓여있는 구조(아이템의 중복 허용)
        - 사용법
            - ArrayList<String> names = new ArrayList<String>();
            - names.add("홍길동");
            - names.add("한석봉");
            - names.add("신사임당");
            - system.out.println(names.get(1));
            - 크기를 정해두지 않고 요소를 추가할 때마다 크기가 커짐
    - 컬렉션은 기본형(int, double, boolean 등)을 취급할 수 없다.
        - ArrayList<int> X
        - 사용법
            - ArrayList<Integer> names = names = new ArrayList<Integer>();
            - names.add(10);
            - names.add(20);
            - names.add(30);
    - Set
        - 순서가 없는 집합
        - 중복 값을 허용하지 않는 집합
        - get() 메서드는 제공하지 않기 때문에 반복이 필요하면 Inerator, foreach 사용
        - List의 contains 보다 압도적으로 빠름
            - 사용법
                - Set<String> colors = new HashSet<>();
                - colors.add("red");
                - colors.add("green");
                - colors.add("red"); // 중복
                - System.out.println(colors.size());
    - Map
        - 키(Key)와 값(Value)의 쌍으로 저장
        - 키의 중복 불가
        - 사용법
            - Map<String, Integer> cities = new HashMap<>();
            - cities.put("서울시", 977);
            - cities.put("수월시", 124);
            - cities.put("부천시", 342);
            - int seoul = cities.get("서울시");
            - System.out.println("서울시 인구는 " + seoul + "만");
            - cities.remove("서울시"); // 삭제
            - cities.put("수월시", 130); // 갱신
            - System.out.println("수원시 인구는 " + cities.get("수원시") + "만");
    - 컬렉션 선택 기준
        - key, value 쌍: Map
        - 중복 가능: List
        - 중복 불가: Set
        - 순서 중요: List
        - 순서 안 중요: Set
        - 검색 속도 중요: Set

## 기억할 것

- List, Set, Map 의 차이
- 되도록 작성을 필드 - 생성자 - getter / setter 순서로 추천

## 어려웠던 점

- for문을 클래스 바로 아래에 작성해서 오류가 났고, 실행 코드는 반드시 메서드 안에 작성해야 한다는 점을 알게 됨
- Person 객체에서 getName() 과 getAge()로 값을 꺼내서 Map에 저장하는 게 헷갈림
- HashMap과 LinkedHashMap의 차이

## 해결 방법

- Person 객체의 getName()은 Key, getAge()는 Value로 생각하고,
  people.put(person.getName(), person.getAge()) 형태로 저장하면 된다는 것을 알게 됨
- MashMap은 출력 순서가 보장되지 않고, LinkedHashMap은 값을 넣은 순서를 유지
