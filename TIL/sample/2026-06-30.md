# 2026-06-30 학습 일지 (인스턴스의 기본 조작)

## 오늘 배운 내용

- **`toString()` 재정의**: 객체의 내부 데이터를 사람이 읽기 쉬운 문자열로 변환하여 출력해 주며, 디버깅 편의성을 위해 필수적으로 재정의해야 함을 배웠습니다.
- **객체의 동등성 비교 규칙 (`equals` / `hashCode`)**:
    - 물리적 주소 비교(`==`) 대신, 객체가 가진 '논리적 데이터 상태'가 같은지 비교하도록 `equals()`를 오버라이딩했습니다.
    - `equals()` 결과가 같다면 `hashCode()`도 반드시 같은 정수값을 반환해야 한다는 **해시 계약(Hash Contract)**을 이해하고, 두 메서드를 항상 세트로 구현했습니다.
- **인스턴스 복사 (`Cloneable`)**: 자바에서 복사 기능을 허용하려면 `Cloneable` 인터페이스를 구현(implements)해야 하며, 기본적으로 얕은 복사(Shallow Copy)가 수행되는 구조를 파악했습니다.

## 기억할 것 (스스로 찾아본 심화 내용)

`publishDate` 데이터가 비어있을 때(`null`)의 방어 코드를 설계하면서, **`if`문(사전 방어)**과 **`try-catch`문(사후 수습)**의 동작 패러다임 차이를 직접 연구하고 비교했습니다.

- **`if`문 방식 (사전 방어 - Look Before You Leap)**
    - 로직을 실행하기 전에 `if (date == null)`처럼 조건을 미리 검사하여 에러를 원천 차단합니다.
    - 값이 없을 가능성이 충분히 예상되는 일반적인 상황(값 검증, 유효성 체크)에 적합하며, 시스템 리소스 소모가 적어 가볍고 빠릅니다.
    - 
- **`try-catch`문 방식 (사후 수습 - It's Easier to Ask Forgiveness Than Permission)**
    - 일단 코드를 질러보고(`try`), 실행 중에 에러(`NullPointerException` 등)가 터지면 그때 낚아채서(`catch`) 우회 도로를 안내합니다.
    - 네트워크 단절, 파일 누락, 외부 서버 다운 등 개발자가 제어하거나 예측할 수 없는 '진짜 예외적인 시스템 상황'을 수습할 때 사용하는 안전장치입니다.

## 실습 코드

슬라이드 결론 규칙을 철저히 준수하면서, 날짜 데이터의 시/분/초를 배제하고 년-월-일로만 객체를 정렬·비교하도록 프로그래밍한 최종 완성 코드입니다.

```java
package com.survivalcoding.day260630;

import java.util.Calendar;
import java.util.Date;
import java.util.Objects;

public class Book implements Comparable<Book>, Cloneable {
    private String title;
    private Date publishDate;
    private String comment;

    // 매직 넘버를 배제하고 의미가 명확한 대문자 상수(Convention) 활용
    private static final int YEAR_NUM = 10000;
    private static final int MONTH_NUM = 100;
    private static final int NULL_DATE_VALUE = 0;

    public String getTitle() { return title; }
    public void setTitle(String title) { this.title = title; }

    public Date getPublishDate() { return publishDate; }
    public void setPublishDate(Date publishDate) { this.publishDate = publishDate; }

    public String getComment() { return comment; }
    public void setComment(String comment) { this.comment = comment; }

    // Date에서 년-월-일만 추출해 8자리 정수로 만드는 메서드 (NPE 발생 시 catch하여 0 반환)
    private int getDate(Date date) {
        try {
            Calendar cal = Calendar.getInstance();
            cal.setTime(date);
            int year = cal.get(Calendar.YEAR);
            int month = cal.get(Calendar.MONTH);
            int day = cal.get(Calendar.DAY_OF_MONTH);
            return (year * YEAR_NUM) + ((month + 1) * MONTH_NUM) + day;
        } catch (NullPointerException e) {
            return NULL_DATE_VALUE;
        }
    }

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (o == null || getClass() != o.getClass()) return false;
        Book book = (Book) o;

        return Objects.equals(title, book.title) &&
                getDate(this.publishDate) == getDate(book.publishDate);
    }

    @Override
    public int hashCode() {
        return Objects.hash(title, getDate(publishDate));
    }

    @Override
    public int compareTo(Book book) {
        int thisDate = getDate(this.publishDate);
        int otherDate = getDate(book.publishDate);

        // 최신 출간일이 앞으로 오도록 정렬 (내림차순)
        return Integer.compare(otherDate, thisDate);
    }

    @Override
    public Book clone() {
        try {
            Book result = (Book) super.clone();
            // 방어 코드: publishDate가 null이 아닐 때만 깊은 복사를 수행하여 NPE 예방
            if (this.publishDate != null) {
                result.publishDate = new Date(this.publishDate.getTime());
            }
            return result;
        } catch (CloneNotSupportedException e) {
            throw new RuntimeException(e);
        }
    }
}
```

## 어려웠던 점

- **제네릭 타입 규격 인식 에러**: `Collections.sort()`로 책 목록을 정렬하려 할 때 제네릭 호환 에러가 발생하여 자바의 정렬 표준 명세와 제네릭 바인딩 구조를 매칭하는 데 혼란이 있었습니다.
- **예외 로직의 실질적 검증**: `getDate()` 내부의 `try-catch`문이 실제로 오작동 없이 작동하여 `null`일 때 `0`을 잘 리턴하는지 테스트 코드 단위에서 확실하게 증명하고 짚고 넘어가는 흐름을 잡기가 어려웠습니다.

## 해결 방법

- 클래스 선언부를 정밀 분석한 결과, 자바 표준 인터페이스 명칭인 `Comparable`을 **`Coparable`로 잘못 타이핑한 오타**를 찾아내어 수정함으로써 제네릭 컴파일 에러를 깔끔하게 해결했습니다.
- JUnit 테스트 코드에서 단순히 무오류를 검증하는 것뿐만 아니라, `null` 객체가 생성한 최종 해시값을 `0`을 대입한 예상값(`Objects.hash(title, 0)`)과 정밀하게 `assertEquals`로 상호 비교함으로써 예외 처리 블록의 완벽한 동작을 입증했습니다.
