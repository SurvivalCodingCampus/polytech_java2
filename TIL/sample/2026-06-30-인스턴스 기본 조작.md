## 오늘 배운 내용

- 모든 클래스는 Object 클래스의 메서드를 가지고 있다.
- Object 타입 변수는 모든 인스턴스를 대입할수 있다.
- 리스트에서 요소 정렬: Collections.sort() 메서드는 컬렉션 내부를 정렬해 줌
- 얕은 복사(Shallow Copy): 객체 자체는 새로 생성, 객체 내의 참조 타입 필드(다른 객체에 대한 참조)는 원본과 동일한 객체를 참조
- 중첨된 객체는 복사되지 않고 주소만 복사, 원본 객체의 참조 타입 필드를 변경하면 복사본의 필드도 같이 변경
- 깊은 복사(Deep Copy): 객체에 담긴 실제 값을 새로운 메모리 공간에 복사, 독립적인 메모리를 가짐(원본이 변화해도 복사본은 영향 없음)

## 기억할 것

- 대표 메서드: ToString()(문자열 표현), equals()(비교), hashCode()(해시 값을 얻음)
- 오버라이드로 원하는 결과를 얻도록 수정 가능

- equals() 메서드를 오버라이드 하여 동등성 규칙 정의가능 List에서 동등성 비교시 사용
- hashCode()를 재정의 하면 Set, Map 내에서의 동등성 규칙으로 사용

- 검색 할때 equals 보다 비용이 싼 hash code 비교를 사용, list는 순차 검색이라 느림
- 모든 객체는 해시값을 가진다
  동일한 객체는 항상 같은 해시값을 가진다.
  하지만, 같은 해시값이라고 항상 동일한 객체는 아니다.

- Collections.sort() 메서드 선언 방식, 정렬대상은 반드시 Comparable 인터페이스를 구현해야 함
- Comparable 인터페이스를 구현하지 않은 객체도 즉석에서 정렬 규칙을 정해 줄수 있다.(CompareTo() 메서드의 규칙과 동일)

```java
//Comparable 인터페이스 예시
public static <T extends Comparable<? super T>> void sort(List<T> list)

//CompareTo() 메서드 예시
public class Account implements Comparable<Account> {
    int number;

    public int compareTo(Account obj) {
        if (this.number < obj.number) { //a와 b의 크기 차이에 따라 변화
            return -1;
        }
        if (this.number > obj.number) {
            return 1;
        }

        return 0;
    }
}

//얕은 복사 예시
public Hero clone() {
    Hero result = new Hero();
    result.name = this.name;
}

//깊은 복사 예시
public Hero clone() {
    Hero result = new Hero();
    result.name = this.name.clone();
}
```
