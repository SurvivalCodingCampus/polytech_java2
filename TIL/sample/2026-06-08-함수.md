# 2026-06-08 함수

## 오늘 배운 내용

- 클래스 : 객체를 만들기 위한 설계도
- 객체 : 클래스를 바탕으로 실제로 만들어 사용하는 대상
- 인스턴스 : `new`를 사용해서 메모리에 만들어진 객체
- 필드 : 객체가 가지고 있는 상태나 값을 저장하는 변수
- 메서드 : 객체가 할 수 있는 동작을 정의한 코드
- 상수 : 값이 변하지 않도록 `static final`로 선언한 값
- `Main` 클래스 : 프로그램을 실행하고 객체를 테스트하는 클래스
- `Cleric` 클래스 : 성직자의 HP, MP와 회복 기능을 정의한 클래스

## 기억할 것

- `Cleric cleric = new Cleric();` : `Cleric` 객체를 만들고 `cleric` 변수에 저장하는 코드
- `name`, `HP`, `MP` : `Cleric` 객체의 상태를 저장하는 필드
- `MAX_HP`, `MAX_MP` : 최대 HP와 최대 MP를 저장하는 상수
- `selfAid()` : MP를 사용해서 HP를 최대치로 회복하는 메서드
- `pray(int sec)` : 기도한 시간 `sec`를 받아 MP를 회복하고, 실제 회복량을 반환하는 메서드
- 매개변수 : 메서드가 실행될 때 외부에서 전달받는 값
- 반환값 : 메서드 실행 후 호출한 곳으로 돌려주는 값
- `return` : 값을 반환하거나 메서드 실행을 중간에 끝낼 때 사용하는 키워드
- 조건문 : HP나 MP가 최대값을 넘지 않도록 검사할 때 사용

## 실습 코드

### Cleric.java

```java
package com.survivalcoding;

public class Cleric {
    static final int MAX_HP = 50;
    static final int MAX_MP = 10;

    String name;
    int HP = MAX_HP;
    int MP = MAX_MP;

    void selfAid() {
        if (HP < 5) {
            return;
        }
        MP -= 5;
        HP = MAX_HP;
    }

    int pray(int sec) {
        int random = (int) (Math.random() * 3);
        int recovery = sec + random;
        int beforeMP = MP;

        MP += recovery;
        if (MP > MAX_MP) {
            MP = MAX_MP;
        }

        return MP - beforeMP;
    }
}
```

## 어려웠던 점

- 클래스와 인스턴스의 차이가 처음에는 헷갈렸다.
- `static final`로 선언한 상수와 일반 필드의 차이를 구분하는 것이 어려웠다.
- `selfAid()`에서 MP를 사용해야 하는데 조건문이 HP를 기준으로 되어 있어서 로직을 더 주의 깊게 봐야 했다.
- `pray(int sec)`에서 회복량을 계산한 뒤, 실제로 회복된 MP를 반환하는 부분이 헷갈렸다.
- `Math.random()`의 결과가 `double`이라서 `int`로 형변환해야 하는 점을 놓치기 쉬웠다.

## 해결 방법

- `Cleric`은 설계도이고 `new Cleric()`으로 만든 `cleric`은 실제 객체라고 정리
- 변하지 않는 최대값은 `MAX_HP`, `MAX_MP`처럼 상수로 두고, 현재 상태는 `HP`, `MP` 필드로 관리한다고 이해
- `Main`에서 메서드 호출 전후의 `HP`, `MP`를 출력해서 객체의 상태가 어떻게 바뀌는지 확인
