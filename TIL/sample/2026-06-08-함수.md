﻿# 2026-06-08 함수

## 오늘 배운 내용

- `Cleric` 클래스를 만들고, `Main` 클래스에서 객체를 생성해 사용하는 흐름을 배웠다.
- 클래스에는 객체의 상태를 나타내는 필드와 객체의 행동을 나타내는 메서드를 작성할 수 있다.
- `HP`, `MP`처럼 객체마다 달라질 수 있는 값은 필드로 만들고, `MAX_HP`, `MAX_MP`처럼 고정된 값은 상수로 만들었다.
- `selfAid()` 메서드를 통해 객체의 MP를 사용하고 HP를 회복시키는 동작을 구현했다.
- `pray(int sec)` 메서드를 통해 매개변수를 받고, 계산 결과를 반환하는 메서드를 구현했다.
- `Main`에서 `new Cleric()`으로 인스턴스를 만들고, 메서드 호출 전후의 상태 변화를 출력해 확인했다.

## 기억할 것

- 클래스는 객체를 만들기 위한 설계도이고, 인스턴스는 클래스로 실제 만들어진 객체이다.
- `Cleric cleric = new Cleric();`은 `Cleric` 객체를 만들고, 그 객체를 `cleric` 변수에 담는 코드이다.
- 필드는 객체의 상태를 저장한다. `name`, `HP`, `MP`가 여기에 해당한다.
- 메서드는 객체의 동작을 정의한다. `selfAid()`와 `pray(int sec)`가 여기에 해당한다.
- `static final`은 변하지 않는 값을 상수로 선언할 때 사용한다.
- 매개변수는 메서드가 실행될 때 외부에서 전달받는 값이다. `pray(int sec)`의 `sec`가 매개변수이다.
- 반환값은 메서드 실행 후 호출한 곳으로 돌려주는 값이다. `pray()`는 실제 회복된 MP 값을 반환한다.
- `return`은 값을 반환할 때도 사용하고, 메서드 실행을 중간에 끝낼 때도 사용할 수 있다.
- HP나 MP처럼 최대값이 정해진 값은 조건문으로 범위를 넘지 않게 처리해야 한다.

## 실습 코드

### Cleric.java

```java
package com.survivalcoding;

public class Cleric {
    static final int MAX_HP = 50;
    static final int MAX_MP = 10;

    String name;
    int HP = MAX_HP;
    int MP = MAX_MP;

    void selfAid() {
        if (HP < 5) {
            return;
        }
        MP -= 5;
        HP = MAX_HP;
    }

    int pray(int sec) {
        int random = (int) (Math.random() * 3);
        int recovery = sec + random;
        int beforeMP = MP;

        MP += recovery;
        if (MP > MAX_MP) {
            MP = MAX_MP;
        }

        return MP - beforeMP;
    }
}
```

## 어려웠던 점

- 클래스와 인스턴스의 차이가 처음에는 헷갈렸다.
- `static final`로 선언한 상수와 일반 필드의 차이를 구분하는 것이 어려웠다.
- `selfAid()`에서 MP를 사용해야 하는데 조건문이 HP를 기준으로 되어 있어서 로직을 더 주의 깊게 봐야 했다.
- `pray(int sec)`에서 회복량을 계산한 뒤, 실제로 회복된 MP를 반환하는 부분이 헷갈렸다.
- `Math.random()`의 결과가 `double`이라서 `int`로 형변환해야 하는 점을 놓치기 쉬웠다.

## 해결 방법

- `Cleric`은 설계도이고 `new Cleric()`으로 만든 `cleric`은 실제 객체라고 정리했다.
- 변하지 않는 최대값은 `MAX_HP`, `MAX_MP`처럼 상수로 두고, 현재 상태는 `HP`, `MP` 필드로 관리한다고 이해했다.
- `Main`에서 메서드 호출 전후의 `HP`, `MP`를 출력해서 객체의 상태가 어떻게 바뀌는지 확인했다.
- `pray()`에서는 `beforeMP`를 저장해 둔 뒤, 회복 후 MP와 비교해서 실제 회복량을 계산한다고 정리했다.
- 최대 MP를 넘지 않도록 `if (MP > MAX_MP)` 조건으로 값을 다시 제한하는 흐름을 확인했다.
