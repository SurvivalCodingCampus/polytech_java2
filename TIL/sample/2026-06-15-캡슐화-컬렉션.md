# 2026-06-15 캡슐화-컬렉션

## 오늘 배운 내용

- 캡슐화 (encapsulation)
    - 클래스나 인스턴스를 이용하여 현실세계를 객체 지향 프로그램으로 자유롭게 개발
    - 실수로 속성을 덮어 쓰거나, 잘못된 조작하는 등의 휴먼 에러를 완전히 없앨 수는 없다.

- 멤버에 대한 액세스 제어로 접근 지정자(access modifier)
    - private: 자기 자신 클래스 안에서만 접근 가능
    - public: 모든 클래스에서도 접근 가능

- 클래스 다이어그램
    - '+' public
    - '-' private

- getter, setter
    - 메소드를 경유한 필드 조작
    - getter: 읽기 전용을 구현할 때 사용
    - setter: 쓰기 전용을 구현할 떄 사용 (잘 쓰지 않는다고 한다)
    - 예시
    - private String name;
      public String getName(){return name;} // 해당 type은 getter
      public void setName(String name){this.name = name;} // void 는 setter

    - Read Only, Write Only 필드의 실현(자유롭게 설정 할 수 있다.)
    - 필드의 이름 등, 클래스의 내부 설계를 자유롭게 변경 가능
    - 필드의 액세스를 검사 가능하다.

    - setter에서 값의 타당성을 검사


- List: 순서대로 쌓여있는 구조

- 배열과 ArrayList 문법이 비슷하면서 다르다.
    - 배열은 미리 메모리를 확보해야한다.
    - ArrayList .add 를 통해서 메모리를 추가할 수 있다.
    - ArrayList 가 메모리를 추가와 삭제를 편하게 할 수 있다.

    - 읽고 쓰는 것은 배열이 빠르지만, 추가와 삭제는 ArrayList가 빠르다.

- ArrayList<int> 는 안됨 <Integer> 로 해야한다.

- Set 은 중복값은 삭제하고 덮어쓰기를 한다.

- Map: 키(key), 값(value)의 쌍으로 이루어진 요소를 담는 자료구조, 키의 중복은 허용되지 않는다.

- List의 for문은 일반 for 문과 다르다.
  List<PersonList> personLists = new ArrayList<>();
  personLists.add(new PersonList("홍길동", 20));
  personLists.add(new PersonList("한석봉", 25));

        for (PersonList name : personLists) {
            System.out.println(name.getName() + "의 나이는 " + name.getAge() + "살");
        }

## 기억할 것

- private를 했다면 getter와 setter를 해야 다른 클래스에서 사용 가능하다.
- 배열과 List, Set, Map은 비슷하지만 다름을 인지해야한다.

```java
void PersonList() {
    List<PersonList> personLists = new ArrayList<>();
    personLists.add(new PersonList("홍길동", 20));
    personLists.add(new PersonList("한석봉", 25));

    for (PersonList name : personLists) {
        System.out.println(name.getName() + "의 나이는 " + name.getAge() + "살");
    }
}
```

- 그 외 오늘 배운 내용을 메모 한 것을 기억해야 한다.
