## 오늘 배운 내용

- Java는 대표적인 객체 지향 언어다.
- 객체는 말그대로 현실의 사물을 뜻하는데 현실의 사물을 코드로 옮기듯이 데이터와 기능을 하나로 묶어서 다룬다.

- 함수는 여러 줄의 코드를 하나의 이름으로 묶어 재사용할 수 있게 해준다.
- Java에서는 클래스 안에 함수를 작성하며, 보통 메서드라고 부른다.
- 함수와 메서드의 차이
- 함수는 인풋이 동일 하면 아웃풋도 동일하지만 메서드는 인풋 뿐 아니라 객체 내부에 저장된 요인에 따라 출력이 달라 질 수 있다.(게임으로 비유하면 플레이어의 공격력에 따라 데미지 값이 달라지는 것)
- 보통 함수와 메서드를 동일시 하는 경우가 많아서 동의어로 사용된다.
- 함수는 입력값을 받을 수 있고, 실행 결과를 반환할 수 있다.
- 같은 동작을 여러 번 작성하지 않고 함수로 분리하면 코드가 더 읽기 쉬워진다.

## 기억할 것

- int a = 15; 에서 타입은 인트, 변수 이름은 a, 리터럴은 15 리터럴은 변수에 들어가는 내용

- 함수 이름은 동작을 잘 나타내도록 작성하는 것이 좋다.
- 클래스 맨 처음은 대문자 Hero
- 필드 최초 이외의 단어 처음 대문자 levelPoint
- 메서드 최초 이외의 단어 처음 대문자 attackPoint

- 클래스를 인스턴스로 생성, 생성한 인스턴스를 새로운 변수 타입이 이용 가능
- 클래스를 정의 하면 같은 타입의 변수 생성 가능

- 매개변수는 함수가 실행될 때 외부에서 받는 값이다.
- 반환값이 없는 함수는 `void`를 사용한다.
- 반환값이 있는 함수는 반환 타입을 적고, `return`으로 값을 돌려준다.
- 함수는 한 가지 역할에 집중하도록 만드는 것이 좋다.

## 실습 코드

```java
// Cleric 클래스

public class Cleric {
    String name = "Cleric"; //성직자 이름
    int hp = 50; //hp
    final int mhp = 50; //최대 hp(final 로 상수 표시)
    int mp = 10; //mp
    final int mmp = 10; //최대 mp(final로 상수 표시)

    void selfAid() { //셀프 에이드, mp를 5 소모 하고 최대 체력 만큼 회복
        mp -= 5;
        hp = mhp;
        System.out.println("slef AId");
        System.out.println(hp);
    }

    void pray(int prayTime) { //기도, 랜덤으로 mp 회복, 기도 시간에 따라 mp 회복량 증가

        prayTime = 0;
        int randomMP = (int) ((Math.random() * 2) + prayTime);
        mp += randomMP;
        System.out.println("pray");
        if (mp >= mmp) {
            System.out.println("max mp");
            mp = mmp;
        }
        System.out.println(mp);
    }
}

//ClericTest 클래스
import org.junit.jupiter.api.DisplayName;
import org.junit.jupiter.api.Test;

import static org.junit.jupiter.api.Assertions.assertEquals;

class ClericTest {

    @Test
    @DisplayName("mp -5 & hp max heal")
    void add() {
        //given
        Cleric cleric = new Cleric();
        //when
        System.out.println(cleric.name); //Cleric
        cleric.selfAid(); //hp max
        cleric.pray(3); //5~8
        cleric.pray(5); //10

        //then
        assertEquals(10, cleric.mp); //10
    }
}
```

## 어려웠던 점

- 메서드 선언 및 내부 코드 작성이 힘들었다
- main 클래스에서 다른 클래스를 불러 오는 것이 힘들었다.
- pray 메서드에서 시간과 mp 회복을 랜덤으로 만드는 것이 힘들었다.
- 한글 출력을 못하는 문제가 발생 했다.

## 해결 방법

- 직접 작성 하면서 메서드 선언 및 출력이 어디 까지 가능한지 실습하였다. (무리하게 이름 변경 시도를 하였고 오류가 생긴 다는 것을 알게 되었다)
- pray 메서드에서 시간까지 랜덤으로 만들지 않고 main 클래스에서 입력을 받게 시도해 보았다.
- 출력을 영어로 변경 하였다.
