## 오늘 배운 내용

- 캡슐화 : 변수나 메소드들을 캡슐로 감싸서 안보이게 하는 정보 은닉 개념 중 하나
- 접근 지정자
    - private : 자기 자신의 클래스
    - package private : 자신과 같은 패키지에 소속된 클래스
    - protected : 자신과 같은 패키지에 소속되던지, 자신을 상속받은 자식 클래스
    - public : 모든 클래스
- getter : 읽기 전용을 구현할 때 사용
- setter : 쓰기 전용을 구현할 때 사용 (잘 안 씀)
- getter과 setter의 메리트
    - Read Only, Write Only 필드의 실현
    - 필드의 이름 등 클래스의 내부 설계를 자유롭게 변경 가능
    - 필드로의 액세스를 검사 가능
