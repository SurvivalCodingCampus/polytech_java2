## 오늘 배운 내용

- 가상세계 = 컴퓨터의 메모리 영역
- 인스턴스 = heap 영역 안에 확보된 메모리(heap 은 동적 할당)
- java 에는 int, double, float, bool, char 등 값을 저장하는 기본형 (primitive type)과 Hero 와 같이 new 로 생성되는 인스턴스인 “레퍼런스 타입" 이 있다
  String 같은 경우는 레퍼런스 타입이다
- 클래스형의 참조
  클래스 형의 변수안엔 인스턴스의 정보가 담긴 메모리 주소가 들어있다.
  어떤 클래스형 변수는 다른 변수에 대입하면 주소 값만 복사된다.
  클래스형은 필드나 메소드의 인수, 리턴 값의 형으로서도 이용된다
- 생성자
  클래스명과 동일 명칭으로 리턴 값의 타입이 명시 되지 않은 메소드는 생성자로 사용
  생성자는 new에 의한 인스턴스화 직후 자동적으로 실행
  생성자는 오버로드에 의한 복수 정의
  클래시에 생성 정의가 1개도 없는 경우 컴파일러가 인수 없음, 처리 내용 없음의 기본 생성 자를 자동 정의
  this()사용 으로 동일 클래스의 다른 생성자를 호출 가능
- 정적 멤버
  ststic 키워드라 붙어 있는 정적 멤버(필드 또는 메서드)
  각 인스턴스가 아닌 클래스가 실체
  클래스 명, 멤버 명, 인스턴스 변수 며, 멤버명은 같은 실체에 액세스
  인스턴스를 1개도 생성하지 않아도 이용 가능
  정적 메서드는 내부에 정적인 메서드와 필드만 이용 해야 한다.
-

## 기억할 것

```java
Hero hero1 = new Hero();

hero1.hp =100;
Hero hero2 = hero1;
hero2.hp =200;
```

- 위 코드 같은 경우엔 hero1과 hero2가 같은 new Hero(); 객체를 보고 있다. 따라서 hero2.hp = 200; 으로 인해, hero1.hp의 값도 똑같이 200이다.
