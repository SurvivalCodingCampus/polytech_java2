# 2026-06-09 클래스

## 오늘 배운 내용

- 가상세계 = 컴퓨터의 메모리 영역
- 인스턴스 = heap 영역 안에 확보된 메모리

## 기억할 것

- Java에는 int,double,float,bool,char등 값을 저장하는 기본형(primitive type)과 Hero와 같이 new 로 생성되는 인스턴스인 "레퍼런스 타입"이 있다
- String은 레퍼런스 타입이다

## 정리

### 클래스형의 참조

- 클래스형 변수의 안에는 “인스턴스의 정보가 담겨있는 메모리 번지"가 들어 있다
- 어떤 클래스형 변수를 다른 변수에 대입하면, 번지정보만 복사 된다.
- 클래스형은 필드나 메소드의 인수, 리턴 값의 형으로서도 이용된다

### 생성자

- “클래스명과 동일 명칭으로, 리턴 값의 타입이 명시되어 있지 않은 메소드"는 생성자로 사용된다.
- 생성자는 new 에 의한 인스턴스화의 직후에 자동적으로 실행 된다.
- 인수를 가지는 생성자를 정의하면, new 를 할 때에 인수를 지정하여 생성자를 실행할 수 있다
- 생성자는 오버로드에 의한 복수 정의가 된다.
- 클래스에 생성자 정의가 1개도 없는 경우에 한해, 컴파일러가 “인수없음, 처리내용없음"의 기본 생성자를 자동정의 해 준다.
- this() 를 사용하면, 동일 클래스의 다른 생성자를 호출 할 수 있다

### 정적 멤버

- static 키워드가 붙어 있는 정적 멤버(필드 또는 메소드) 는
    - 각 인스턴스가 아닌, 클래스에 실체가 준비된다.
    - ”클래스명, 멤버명", “인스턴스 변수명, 멤버명" 의 어디에도 같은 실체에 액세스하게 된다
    - 인스턴스를 1개도 생성하지 않아도 이용 가능
- 정적 메소드는 그 내부에 정적이지 않은 메소드나 필드를 이용하는 것이 불가능하다

## 실습 코드

```java
public Cleric(String name) {
    this.name = name;
    this.HP = MAXHP;
    this.MP = MAXMP;
}

public Cleric(String name, int hp) {
    this.name = name;
    this.HP = hp;
    this.MP = MAXMP;
}

public Cleric(String name, int hp, int mp) {
    this.name = name;
    this.HP = hp;
    this.MP = mp;
}

```

## 어려웠던 점

- 테스트를 진행할때 void 테스트_이름(){...} 형태의 메소드로 코드를 감싸줘야 에러가 안나고 어노테이션은 메소드 바로 위에 붙여야한다는것을 몰라 왜 안되는지 인터넷을 찾아봤다.
