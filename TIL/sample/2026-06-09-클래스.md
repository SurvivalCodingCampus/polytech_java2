## 오늘 배운 내용

- 가상 세계 = 컴퓨터의 메모리 영역
    - 메모리 - 휘발성
- 인스턴스 = heap 영역 안에 확보된 메모리
    - heap : 동적 메모리 할당을 위한 공간 (창고)
    - Stack : 데이터를 저장하는 가장 기본적인 자료 구조 중 하나 (작업대) / 지역변수
- 타입
    - 기본형(primitive) : int, double, bool, char
    - 참조형(reference) : new (String)
- 필드의 초기 값
    - 배열과 String은 null
- 클래스 형의 참조
    - 인스턴스의 정보가 담겨있는 메모리 번지
    - 어떤 클래스 형 변수를 다른 변수에 대입하면 번지 정보만 복사
- 생성자
    - 리턴 값의 타입이 명시되어 있지 않은 메소드 (클래스명과 동일)
    - new 에 의한 인스턴스화 직후 자동적으로 실행
    - 오버로드에 의한 복수 정의
    - this(자기자신) 사용 시, 동일 클래스의 다른 생성자 호출
- 정적 멤버 (static)
    - 각 인스턴스가 아닌 클래스에 실체가 준비
    - 클래스명, 멤버명, 인스턴스 변수명 의 어디에도 같은 실체에 엑세스
    - 인스턴스를 1개도 생성하지 않아도 이용 가능
- 테스트 방법
    - 수동 테스트 (사람이 스스로 테스트)
    - 단위 테스트 (1개의 함수 테스트)
    - 통합 테스트 (여러개의 클래스나 함수를 같이 테스트)
- 화이트 박스 테스트 : 내부 구조와 동작에 중점을 두고 테스트하는 방법 (구문검사, 경로검사, 조건/분기 검사)
- 블랙 박스 테스트 : 소프트웨어 내부 구조를 무시하고 기능을 테스트하는 방법 (등가 분할, 경계 값 분석, 상태 전이 테스트)
- 테스트의 장점
    - 신속한 피드백
    - 조기 장애 감지
    - 더 안전한 리펙토링
    - 안정적인 개발 속도

## 기억할 것

- 클래스, 인스턴스, 필드, 생성자, 정적 멤버의 대한 정의
- 생성자 이름은 클래스 이름과 반드시 같아야 함
- 생성자 오버로드는 매개변수 개수나 타입을 다르게 해서 생성자를 여러 개 만드는 것
- new Cleric() 같은 경우 이름이 없기 때문에 인스턴스화 불가능
