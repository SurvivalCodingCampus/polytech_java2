# 2026-06-22 추상 클래스와 인터페이스

## 오늘 배운 내용

- 추상 클래스는 여러 클래스에서 공통으로 가지는 필드나 메서드를 부모 클래스로 묶을 때 사용한다.
- `abstract`가 붙은 클래스는 직접 객체로 만들 수 없다.
- 인터페이스는 클래스가 반드시 가져야 하는 기능을 약속하는 역할을 한다.
- `implements`를 사용하면 인터페이스의 메서드를 구현해야 한다.
- 부모 타입이나 인터페이스 타입으로 자식 객체를 다룰 수 있는데, 이것이 다형성의 기본 느낌이다.

## 기억할 것

- 추상 클래스는 "공통 속성과 기능"을 묶을 때 사용한다.
- 인터페이스는 "이 기능을 반드시 할 수 있어야 한다"는 규칙을 만들 때 사용한다.
- 인터페이스는 생성자를 강제하지 못한다.
- 생성자에서 값을 받을지, setter로 나중에 받을지는 클래스 설계에서 결정한다.
- 부모 생성자의 매개변수가 바뀌면 자식 클래스의 `super(...)` 호출도 같이 바꿔야 한다.

## 실습 코드

```java
public interface Thing {
    double getWeight();

    void setWeight(double weight);
}
```

```java
public abstract class Asset {
    String name;
    int price;

    public Asset(String name, int price) {
        this.name = name;
        this.price = price;
    }
}
```

```java
public abstract class TangibleAsset extends Asset implements Thing {
    String color;
    double weight;

    public TangibleAsset(String name, int price, String color, double weight) {
        super(name, price);
        this.color = color;
        this.weight = weight;
    }

    @Override
    public double getWeight() {
        return weight;
    }

    @Override
    public void setWeight(double weight) {
        this.weight = weight;
    }
}
```

```java
public class Book extends TangibleAsset {
    String isbn;

    public Book(String name, int price, String color, double weight, String isbn) {
        super(name, price, color, weight);
        this.isbn = isbn;
    }
}
```

```java
public class Computer extends TangibleAsset {
    String makerName;

    public Computer(String name, int price, String color, double weight, String makerName) {
        super(name, price, color, weight);
        this.makerName = makerName;
    }
}
```

## 다형성 맛보기

```java
Thing book = new Book("자바의 정석", 30000, "Blue", 0.85, "978-89-000-0000-0");
Thing computer = new Computer("맥북 프로", 2500000, "Silver", 1.45, "Apple");

System.out.println(book.getWeight());
System.out.println(computer.getWeight());
```

- `Book`과 `Computer`는 서로 다른 클래스다.
- 하지만 둘 다 `Thing`을 구현한 `TangibleAsset`을 상속하므로 `Thing` 타입으로 다룰 수 있다.
- 이렇게 같은 타입처럼 다루면서 실제 객체에 따라 동작하게 하는 것이 다형성과 연결된다.

## 어려웠던 점

- 기존에는 클래스를 객체를 만들기 위한 틀이라고 생각했다.
- 예를 들어 클래스가 붕어빵 틀이고, 객체는 그 틀로 찍어낸 붕어빵이라고 이해했다.
- 그런데 추상 클래스는 직접 객체를 만들기 위한 틀이라기보다, 여러 클래스의 공통된 부분만 미리 만들어 두는 부모 틀이라는 점이 헷갈렸다.
- 인터페이스도 추상 클래스처럼 코드의 중복을 줄이고 모듈화를 위해 사용하는 것처럼 보여서 둘의 차이가 잘 구분되지 않았다.
- 추상 클래스는 공통 필드와 공통 메서드를 가질 수 있고, 인터페이스는 어떤 기능을 반드시 구현해야 한다는 약속에 가깝다는 점을 이해하는 데 시간이 걸렸다.

## 해결 방법

- 추상 클래스와 인터페이스에 대해 자료를 더 찾아보면서 둘의 차이를 구분하게 되었다.
- 추상 클래스와 인터페이스는 둘 다 직접 객체를 만들기 위한 용도라기보다, 다른 클래스가 가져야 할 구조나 기능을 정리하기 위해 사용한다.
- 인터페이스는 기본 형틀처럼 메서드의 규칙을 정해 두고, 필요한 클래스에 `implements`로 붙여서 사용할 수 있다.
- 인터페이스를 사용하면 서로 상속 관계가 없는 클래스들도 같은 기능을 가진 타입처럼 다룰 수 있다.
- 추상 클래스는 일반 클래스처럼 필드와 메서드를 가질 수 있지만, 여러 자식 클래스의 공통점을 모아 부모 형태로 만든다는 점이 다르다.
- 자식 클래스는 추상 클래스를 `extends`로 상속받고, 필요한 메서드를 오버라이딩해서 자기 상황에 맞게 구현할 수 있다.
- 오버로딩은 부모 기능을 수정하는 것이 아니라 같은 이름의 메서드를 매개변수만 다르게 추가하는 것이므로, 추상 클래스에서 기능을 바꿀 때는 주로 오버라이딩을 사용한다.
