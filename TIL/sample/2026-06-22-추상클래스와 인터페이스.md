## 오늘 배운 내용

- 추상 클래스 : 상속의 재료로 사용되는 클래스 상세 부분이 일부 미정인 클래스(공통된 뼈대를 만드는 미완성 클래스)
- 인터페이스 : 추상 클래스중 추상 메소드만 가진 것 (클래스가 구현해야 하는 메서드와 상수의 집합)

## 기억할 것

- 추상 클래스는 abstract를 통해서 선언 가능
- 인터페이스는 클래스 대신 interface 선언 다른 클래스에서 사용시 implements를 사용
- 인터페이스 다중 상속 가능
- 인터페이스에 선언된 메소드는 자동적으로 public abstract 가 되고, 필드는 public static final 이 된다
  복수의 인터페이스를 부모로 두는 다중상속이 가능

## 실습 코드

```java
public interface Thing {
    double getWeight();

    void setWeight();

}

public abstract class TangibleAsset extends Asset implements Thing {
    private int price;
    private String name;
    private String color;
    private String AssetType;
    double weight;

    void AssetType() {
        super.setAssetType(AssetType);
    }

    public double getWeight() {
        return weight;
    }

    public void setWeight(double weight) {
        this.weight = weight;
    }

    public void setName(String name) {

        this.name = name;
    }

    public String getName() {
        return name;
    }

    public void setColor(String color) {

        this.color = color;
    }

    public String getColor() {
        return color;
    }

    public void setPrice(int price) {
        this.price = price;
    }

    public int getPrice() {
        return price;
    }
}

public class Book extends TangibleAsset {
    String name;
    int price;
    String color;
    String isbn;

    void price() {
        super.setPrice(price);
    }

    void color() {
        super.setColor(color);
    }

    void name() {
        super.setName(name);
    }

    @Override
    public void setWeight() {

    }
}
```
