# 2026-06-22 추상클래스와 인터페이스

## 오늘 배운 내용

- 추상클래스(abstract) : 클래스 간에 '부모-자식' 관계가 명확할 때
    - 공통적인 필드(name, age 등)와 메서드를 공유하고 싶을 떄 적합
    - 부모가 Animal 자식이 Dog,Cat 등 이라고 볼 수 있다.

    - 상속의 재료로 사용 되는 클래스
    - 상세 부분이 일부 미정인 클래스
    - 추상클래스는 인스턴스화가 금지되어 있다. (new 가 안된다.)


- 인터페이스(Interface) : 관계가 없는 클래스들간에 '동일한 기능 부여' 하고 싶을 때
    - Flyable 즉 날 수 있다. 라는 관계
    - 새, 비행기 등 클래스 간에 관계가 다르지만 날 수 있다라는 기능이 있다.

    - 모든 메소드는 추상 메소드 여야 한다.
    - 필드를 가지지 않는다.
        - public static final 이 생략되어 있어 상수로만 선언이 된다.

    - implement 로 구현 (extends 로 하지 않는다는 점)
    - 인터페이스를 구현한 클래스들은 공통 메소드로 구현이 된다.
    - 여러 인터페이스를 구현할 수 있다.

## 기억할 것

- 추상 클래스는 상세 부분이 일부 미정인 클래스
- 인터페이스는 필드를 가지지 않고 가진다면 상수로만 선언한다.

## 실습 코드

```java
public abstract class Asset {
    private String name;
    private int price;
    private String color;

    public Asset(String name, int price, String color) {
        this.name = name;
        this.price = price;
        this.color = color;
    }

    public String getName() {
        return name;
    }

    public int getPrice() {
        return price;
    }

    public String getColor() {
        return color;
    }
}

public abstract class TangibleAsset extends Asset implements Thing {
    private double weight;

    public TangibleAsset(String name, int price, String color) {
        super(name, price, color);
    }

    public abstract double weight();

    public abstract void setWeight(double weight);
}

public class Book extends TangibleAsset {
    private String isbn;

    public Book(String name, int price, String color, String isbn) {
        super(name, price, color);
        this.isbn = isbn;
    }

    public String getIsbn() {
        return isbn;
    }

    @Override
    public double weight() {
        return 0;
    }

    @Override
    public void setWeight(double weight) {

    }
}

public interface Thing {
    double weight();

    void setWeight(double weight);
}
```
