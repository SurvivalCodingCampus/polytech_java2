## 핵심 기억할 것
- **문자열 결합 방식의 차이점**
    - `+` 연산자: String의 불변성(Immutable) 때문에 매번 새로운 객체를 생성하여 반복문 내 사용 시 메모리 낭비 심함.
    - `StringBuilder`: 가변(Mutable) 객체이며 동기화를 지원하지 않아 단일 스레드에서 가장 빠름.
    - `StringBuffer`: 가변(Mutable) 객체이며 thread-safe(동기화 지원)하여 멀티 스레드 환경에 적합.


- **자주 사용하는 메소드**: `replace()`, `split()`, `length()`, `endsWith()` 등


- **정규표현식의 기본 기호**: `.` (임의의 문자), `*` (0회 이상 반복), `^` (맨 앞), `$` (맨 뒤), `?` (0회 또는 1회)

##  실습 및 테스트 코드

### 1. 1부터 100까지 콤마로 연결 후 배열 생성 (연습문제 1-1)
```java
package com.survivalcoding.day260706;

import java.util.Arrays;

public class Regular_Expression {
    private String[] strArr;

    public void setRegular_Expression() {
        StringBuilder sb = new StringBuilder();
        for (int i = 1; i <= 100; i++) {
            sb.append(i).append(",");
        }
        this.strArr = sb.toString().split(",");
    }

    public void getRegular_Expression() {
        System.out.println(Arrays.toString(strArr));
    }

    public String[] getStrArr() {
        return this.strArr;
    }
}
```

## 어려웠던 점

- StringBuilder와 StringBuffer의 작동 원리
  - `+` 연산자 대신 왜 가변 버퍼 클래스를 써야 하는지 메모리 관점(불변성 vs 가변성)에서 이해하는 부분이 조금 낯설었습니다.

  - 두 클래스가 기능은 비슷한데 멀티 스레드 동기화 여부(Thread-Safe)에 따라 갈린다는 개념이 완전히 와닿지 않아 조사가 필요했습니다.


- 정규표현식 조건 해석 및 작성

  - ^, $, ?, {} 같은 특수 기호들이 여러 개 조합되었을 때, 머릿속으로 한눈에 매칭 조건을 그려내는 것이 생각보다 까다로웠습니다.

## 해결 방법

- 공식 문서 및 자료 조사

    - String은 값이 바뀔 때마다 새로운 객체를 만들어 쓰레기 메모리를 남기지만, StringBuilder는 하나의 메모리 공간을 늘려가며 쓴다는 핵심 차이점을 확실히 정리했습니다.
