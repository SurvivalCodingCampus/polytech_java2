# 2026-06-29 제네릭과 열거형

## 오늘 배운 내용

- 제네릭(Generic)
    - 타입을 나중에 원하는 형태로 정의할 수 있음
    - ☆ 타입 안전(type safe) 효과 : 이 효과 때문에 쓰는 것이다.
    - <E> 로 하는 것이 제네릭이라고 한다. 

    - Object 는 모든 타입을 담을 수 있다?
        - 모든 클래스는 Object 를 상속 받은 것으로 한다.
        - 모든 Type은 Object Type 이다.
        - get 할 때 불편함이 있다.

    - public class Pocket<E extends TangibleAsset>
        - 이렇게 하면 관련된 것만 불러올 수 있다.

- 열거형(enum)
    - 여러가지 Type을 담아서 조건문을 작성할 때 유용

## 기억할 것

- 제네릭을 사용하면 여러 가지 타입을 저장과 꺼내 쓸 수 있다.

=========

## 실습 코드

```java
public class StrongBox<E> {
    private E data;
    private final KeyType keyType;
    private int count = 0;

    public StrongBox(KeyType keyType) {
        this.keyType = keyType;
    }

    public void put(E data) {
        this.data = data;
    }

    public E get() {
        count++;
        if (count >= keyType.getLimit()) {
            return this.data;
        }
        return null;
    }

}
```

## 어려웠던 점

- 열거형을 작성하며 어떻게 작성하면 될지 고민을 하게 되었다.

## 해결 방법

- 생성자, getter 의 작성
