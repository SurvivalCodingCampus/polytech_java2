## 오늘 배운 내용

### 1. 클래스형의 참조
  - 클래스형 변수 안에는 "인스턴스의 정보가 담긴 메모리 번지가 포함"된다.
  - 어떤 클래스형 변수를 다른 변수에 대입하면, 번지정보만 복사된다.
  - 클래스형 필드나 메소드의 인수, 리턴 값의 형으로서도 이용된다.
  
### 2. 생성자
- 클래스명과 동일한 명칭이면서, 리턴 값의 타입이 명시되어 있지 않은 메서드는 생성자로 사용된다.
- 생성자는 `new`에 의한 인스턴스화 직후에 자동적으로 실행된다.
- 인수를 가지는 생성자를 정의하면, `new`를 할 때 인수를 지정하여 생성자를 실행할 수 있다.
- 생성자는 오버로딩에 의해 복수 정의(오버로딩)가 가능하다.
- 클래스에 생성자 정의가 1개도 없는 경우에 한해, 컴파일러가 “인수 없음, 처리 내용 없음"의 기본 생성자(Default Constructor)를 자동 정의해 준다.
- `this()`를 사용하면, 동일 클래스의 다른 생성자를 호출할 수 있다.

### 3. 정적 멤버
- <mark style="background-color: #BFF3FF">static</mark> 키워드가 붙어 있는 정적 멤버(필드 또는 메소드)는
    1. 각 인스턴스가 아닌, 클래스 영역에 실체가 준비된다.
    2. "클래스명.멤버명", “인스턴스변수명.멤버명" 어느 쪽으로 접근해도 모두 같은 실체에 액세스하게 된다.
    3. 인스턴스를 1개도 생성하지 않아도 이용 가능하다.
    - **주의:** 정적 메소드는 그 내부에 정적이지 않은(Instance) 메소드나 필드를 이용하는 것이 불가능하다. (스static 영역이 메모리에 먼저 올라가기 때문)

## 기억할 것

- 오버로딩(Overloading)
  - 개념: 한 클래스 내에 <mark style="background-color: #BFFFBB">이름이 같은 메서드를 여러 개 만드는 것.</mark>
  - 핵심 조건: 메서드 <mark style="background-color: #BFFFBB">이름은 같아야 하지만, 매개변수의 타입, 개수, 순서 중 하나는 달라야</mark> 합니다. (리턴 타입만 다른 것은 인정되지 않습니다.)```java
``` java
  class Cleric {
      // 1번: 매개변수가 없는 공격 (기본 공격)
      void attack() {
          System.out.println("지팡이로 때립니다.");
      }

      // 2번: 대상이 있는 공격 (이름이 같지만 매개변수가 다름!)
      void attack(String monster) {
          System.out.println(monster + "에게 신성 마법을 쏩니다.");
      }
  }
```
- 오버라이딩(Overriding)
  - 개념 : 상속 관계에서 부모 클래스가 가진 메서드를 <mark style="background-color: #FFB7B7">자식 클래스가 맘에 들지 않아 똑같이 다시 짜는 것.</mark>
  - 핵심 조건: 부모 메서드와 이름, 매개변수, 리턴 타입이 <mark style="background-color: #FFB7B7">100% 똑같아야 합니다.</mark>
```java
// 부모 클래스
class Hero {
    void run() {
        System.out.println("영웅이 전력질주로 뛰어갑니다.");
    }
}

// 자식 클래스
class Cleric extends Hero {
    // 오버라이딩: 부모의 run()이 맘에 안 들어서 내 방식대로 바꿈!
    @Override
    void run() {
        System.out.println("성직자는 둔해서 느릿느릿 걸어갑니다.");
    }
}
```
## 실습 코드

```java
package com.survivalcoding.day260609;

import java.util.Random;

public class Cleric {
    static final int maxHp = 50;
    static final int maxMp = 10;
    int hp;
    int mp;
    String name;


    Cleric(String name){
        this(name, maxHp, maxMp);
    }

    Cleric(String name, int hp){
        this(name, hp, maxMp);
    }

    Cleric(String name, int hp, int mp){
        this.name = name;
        this.hp = hp;
        this.mp = mp;
    }

    void selfAid() {
        this.mp -= 5;
        this.hp = maxHp;
    }

    int pray(int sec) {
        Random point = new Random();
        int randomPoint = point.nextInt(sec, sec + 3);
        int realRecovery = randomPoint;

        if (this.mp + randomPoint > maxMp) {
            realRecovery = maxMp - this.mp;
        }

        this.mp += realRecovery;

        System.out.println(this.name + "는 mp를 " + realRecovery + "만큼 회복하였다.");

        return realRecovery;
    }
}

```

## 어려웠던 점

- 기존에 `this.변수명` 이렇게 많이 이용하였는데 이번에 공부하면서 `this(변수1, 변수2 ...)`로 가능한 것을 알게되었다.
  - this.변수 : 필드에 값 직접 대입
  - this(변수1, 변수2 ...) : 다른 생성자 호출 ※ 이 구문은 반드시 생성자 안에 <mark>첫 번째 줄</mark>에만 사용할 수 있다.
