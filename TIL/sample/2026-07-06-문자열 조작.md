# 2026-07-06 문자열 조작

## 오늘 배운 내용

- 자바의 문자열(String)은 불변(Immutable)의 특성을 가지기 때문에, 문자열을 변경하는 메서드들은 기존 문자열을 바꾸는 것이 아니라 언제나 새로운 문자열 객체를 반환한다.
- 자바에서 문자열 처리 유형
- ① 문자열 결합(Concatenation): 여러 문자열을 하나로 합치는 작업이다.
-
    + 연산자: 가장 직관적이고 흔히 쓰이는 방법이다.
- concat() 메서드: 기존 문자열 뒤에 다른 문자열을 이어 붙인다.
- 반복문 안에서 문자열을 여러 번 합칠 때는 +대신 StringBuilder나 StringBuffer를 사용하는 것이 성능(메모리)상 훨씬 유리하다.
- ② 일부 떼어내기(Substring): 문자열의 특정 일부분만 추출할 때 사용한다.
- 자바의 인덱스는 0부터 시작하며, 끝 인덱스는 포함되지 않는다.((시작 인덱스, 끝 인덱스))
- substring(int beginIndex): 시작 인덱스부터 끝까지 잘라낸다.
- substring(int beginIndex, int endIndex): 시작 인덱스부터 endIndex - 1까지 잘라낸다.
- ③ 일부 치환(Replace): 문자열 내의 특정 문자나 패턴을 다른 문자로 바꿀 때 사용한다.
- replace(CharSequence target, CharSequence replacement): 일치하는 모든 문자열을 바꾼다.
- replaceAll(String regex, String replacement): 정규 표현식(Regex)을 사용하여 일치하는 모든 문자열을 바꾼다.
- replaceFirst(String regex, String replacement): 정규 표현식과 일치하는 첫 번째 부분만 바꾼다.
- ④ 분리(Split): 구획 문자(Delimiter)를 기준으로 문자열을 쪼개어 배열로 만들 때 사용한다.
- split(String regex): 정규 표현식을 기준으로 문자열을 분리하여 String[] 배열로 반환한다.
- ⑤ 대소문자 변경(Case Conversion): 영문 문자열을 모두 대문자 또는 소문자로 변환할 때 사용한다.
- toUpperCase(): 모든 문자를 대문자로 변환한다.
- toLowerCase(): 모든 문자를 소문자로 변환한다.
- ⑥ 검색(Search): 문자열 내에 특정 문자나 문자열이 포함되어 있는지, 혹은 어디에 위치하는지 찾을 때 사용한다.
- indexOf(String str): 앞에서부터 검색하여 지정한 문자열이 처음 나타나는 인덱스를 반환한다.(없으면 -1)
- lastIndexOf(String str): 뒤에서부터 검색하여 가장 마지막 인덱스를 반환한다.
- contains(CharSequence s): 특정 문자열이 포함되어 있는지 여부를 boolean으로 반환한다.
- startWith(String prefix) / endsWith(String suffix): 특정 문자열로 시작하는지 / 끝나는지 확인한다.
- ⑦ 내용 비교(Comparison): 두 문자열의 실제 텍스트 내용이 같은지 비교할 때 사용한다.
- 자바에서는 문자열 비교 시 == 연산자가 아닌 메서드를 사용해야 한다.
- == 연산자는 주소값(참조)을 비교하므로, 실제 글자가 같은지 비교할 때는 반드시 아래 메서드들을 써야 한다.
- equals(Object anObject): 대소문자를 구분하여 내용이 같은지 비교한다.
- equalsIgnoreCase(String anotherString): 대소문자를 구분하지 않고 내용이 같은지 비교한다.
- ⑧ 길이(Length): 문자열의 총 글자 수(길이)를 구할 때 사용한다.
- length(): 문자열의 길이를 정수(int)로 반환한다.
- 공백도 글자 수에 포함된다.
- ⑨ 변환(Type Conversion): 다른 자료형(숫자 등)을 문자열로 바꾸거나, 반대로 문자열을 다른 자료형으로 바꿀 때 사용한다.
- 다른 자료형 -> 문자열: String.valueOf()메서드를 사용한다.
- 문자열 -> 다른 자료형: 각 데이터 타입의 Wrapper 클래스에 있는 parse...() 메서드를 사용한다.
- 이 외에도 문자열 앞뒤 공백을 제거해 주는 trim()이나 strip() 메서드도 실무에 많이 쓰인다.
- 자바에는 int, double, boolean처럼 메모리 효율을 위해 사용하는 기본 데이터 타입(Primitive Type)이 있다.
- 기본 데이터 타입은 아주 빠르고 가볍지만, '객체(Object)'가 아니라서 메서드(기능)를 가질 수 없다는 한계가 있다.
- 그래서 자바는 이 기본 타입들을 객체로 감싸서(Wrap) 더 다양하게 활용할 수 있도록 클래스를 만들어 두었는데, 이를 Wrapper(래퍼) 클래스라고 부른다.
- 한 마디로, 포장지를 씌운다고 생각하면 된다.
- 기본 타입(Primitive Type)과 Wrapper Class의 표현법 차이
- 기본 타입에서는 int로 표현하지만, Wrapper 클래스에서는 Integer로 표현한다.
- 기본 타입에서는 char로 표현하지만, Wrapper 클래스에서는 Character로 표현한다.
- 위의 두 가지 예시를 통해, Wrapper 클래스는 표현할 때, 첫 글자는 대문자로 표현하고 전체 다 적는 것을 알 수 있다.
- Wrapper 클래스를 사용하는 이유
- 단순한 숫자 변환 기능(메서드)을 제공하는 것뿐만 아니라, 자바의 중요한 컬렉션(예: ArrayList<Integer>)에는 기본 타입(int)을 담을 수 없고 오직 객체만 담을 수 있기 때문에 Wrapper
  클래스가 반드시 필요하다.
- parse라는 단어는 "구문 분석하다, (원하는 형태로)해석하다"라는 뜻을 가진다.
- 즉, parse...() 메서드는 문자열을 읽어서 해당 데이터 타입으로 해석해 변환해 주는 기능을 가지고 있다.
- Integer.parseInt(String s): 문자열을 int형 정수로 변환.
- Double.parseDouble(String s): 문자열을 double형 실수로 변환.
- Boolean.parseBoolean(String s): 문자열을 boolean형 논리값으로 변환.
- 사용할 때 주의할 점(NumberFormatException)
- parse...() 메서드는 문자열 안에 정말로 숫자로 바꿀 수 있는 글자가 들어있을 때만 정상 작동한다.
- 만약 숫자로 바꿀 수 없는 문자열을 넣으면 자바에서 NumberFormatException 이라는 에러(예외)를 발생시킨다.
- 따라서 실무에서는 웹 페이지 창 등에서 사용자가 값을 입력할 때, 숫자가 맞는지 미리 검증(Validation)한 후에 이 메서드들을 사용한다.
- isEmpty(): 문자열의 길이가 0인지 확인할 때 사용한다.
- 자바에서 StringBuilder와 StringBuffer는 모두 문자열을 동적으로 변경(추가, 수정, 삭제)할 수 있도록 도와주는 클래스이다.
- StringBuilder와 StringBuffer는 모두 가변(Mutable)이다.
- StringBuilder는 동기화(Synchronization)를 지원 안 하지만(Not Thread-Safe), StringBuffer는 지원한다.(Thread-Safe)
- StringBuilder는 연산 속도가 가장 빠르지만, StringBuffer는 보통이다.
- StringBuilder는 추천 사용 환경이 단일쓰레드 환경(일반적인 경우)이지만, StringBuffer는 멀티쓰레드 환경이다.
- StringBuilder와 StringBuffer 각각의 특징과 장점 그리고 단점
- StringBuilder: 불안전하지만 엄청 빠르다.(Not Thread-Safe)
- StringBuilder는 동기화 처리를 하지 않아서 여러 쓰레드가 동시에 달려들면 데이터가 깨질 위험이 있다.
- 장점: 동기화 과정을 건너뛰기 때문에 문자열 연산 속도가 가장 빠르다.
- 단점: 멀티쓰레드 환경에서 안전하지 않다.
- StringBuffer: 안전하지만 조금 느리다.(Thread-Safe)
- StringBuffer는 메서드마다 sychronized 키워드가 붙어 있다.
- 이는 여러 쓰레드가 동시에 이 객체에 접근해서 문자열을 수정하려고 해도, 한 번에 하나의 쓰레드만 안전하게 처리하도록 제어(동기화)해 준다는 뜻이다.
- 장점: 멀티쓰레드 환경에서 데이터가 꼬이지 않고 안전하다.
- 단점: 동기화 처리를 위한 락(Lock)오버헤드 때문에 StringBuilder보다 속도가 조금 느리다.
- 실제로 우리가 작성하는 대부분의 프로그램(메서드 내부의 지역 변수 등)은 단일 쓰레드 안에서 동작한다.
- 따라서 대부분의 개발 상황에서는 성능이 우수한 StringBuilder를 사용하는 것이 기본 권장 사항이다.
- StringBuilder: append() 메서드로 결합한 결과를 내부 메모리(버퍼)에 담아 두고 toString()으로 결과를 얻는다.
- 메서드 체인: append() 처럼 자기 자신의 참조를 리턴하여 연속해서 메서드를 호출하는 기법.
- 정규표현식(Regular Expression): 특정한 규칙을 가진 문자열의 집합을 표현하는 데 사용하는 형식 언어이다.
- 매칭할 문자 지정
- . : 임의의 한 문자.(줄바꿈 제외)
- ^ : 문자열의 시작을 의미.
- $ : 문자열의 끝을 의미.
- 문자 클래스(대괄호 [])
- [abc]: a, b, c 중 하나.
- [^abc]: a, b, c를 제외한 모든 문자.(^가 대괄호 안에 있으면 부정의 의미)
- [a-z]: 알파벳 소문자 전체.
- [A-Z]: 알파벳 대문자 전체.
- [0-9]: 숫자 전체.
- 자주 쓰는 단축 클래스
- \d: 숫자.([0-9]와 동일)
- \D: 숫자가 아닌 것.([^0-9]와 동일)
- \w: 알파벳 대문자, 소문자, 숫자, 언더바(_).([a-zA-Z0-9_]와 동일)
- \W: 알파벳, 숫자, 언더바가 아닌 것.
- \s: 공백 문자.(스페이스, 탭 등)
- 수량자 (반복 횟수 지정)
-
    * : 0번 이상 반복.
-
    + : 1번 이상 반복.
- ? : 0번 또는 1번만 존재.
- {n}: 정확히 n번 반복.
- {n,}: 최소 n번 이상 반복.
- {n,m}: n번부터 m번까지 반복.
- 문자열 포맷(String Formatting)은 특정 형식(틀)을 미리 만들어 두고, 그 안에 원하는 데이터(변수 값)를 동적으로 삽입하여 새로운 문자열을 만드는 방법이다.
- 자바에서는 주로 String.format() 메서드나 System.out.printf()를 사용한다.
- 문자열 포맷을 사용하면 문장의 전체적인 틀을 한 눈에 볼 수 있어 가독성이 훨씬 좋아진다.
- % [기호] [갯수] [형태]
- %: "이제부터 포맷 지정을 시작한다"는 신호이다.
- [기호] (생략 가능): 정렬 방식이나 콤마, 0을 채울지 등 꾸미기 옵션이다.
- , : 금액 표시할 때처럼 3자리마다 쉼표를 찍어준다.
- 0 : 공백이 생기면 빈칸 대신 숫자 0으로 채워준다.
-
    - : 기본 우측 정렬인 출력을 왼쪽으로 붙여서 정렬한다.
-
    + : 양수일 때도 부호(+)를 강제로 표시한다.
- [갯수] (생략 가능): 화면에 차지할 전체 글자 칸수나 소수점 자리수를 지정한다.
- 예를 들면 5라고 적으면 무조건 5칸을 확보하고, 5.2라고 적으면 전체 5칸을 확보하고, 소수점은 밑으로 2자리까지만 보여달라는 뜻이다.
- [형태] (필수): 어떤 종류의 데이터(데이터 타입)를 넣을지 결정한다.
- d(정수), s(문자열), f(실수), b(불리언).
