# 2026-06-30 자바 학습 정리 : 인스턴스 기본조작

## 오늘 배운 내용

- Objects : equals, hashCode
- Set, Map
- Collections.sort
- 얕은 복사, 깊은 복사

## 기억할 것

- 기본 equals는 주소 값을 기반으로 비교한다.
    - 오버라이드를 사용하여 동등성 규칙을 객체의필드 값 등으로 재정의한다.
- 해시코드도 objects의 필드등을 이용하여 재정의하여 값이 같은 것들은 같은 해시값을 가지게 한다.
- Comparable 인터페이스를 활용하여 compareTo를 재정의 하여 정렬 기준을 재정의한다.
- 얕은 복사는 메모리 주소를 참조하여 사용하는 것 , 깊은 복사는 새로운 메모리 주소를 생성한 뒤 값을 복사하는 것

## 실습 코드 (테스트 구조 검증)

```Java
public class BookTest {
    private Book book;
    private Book book2;
    private Book book3;

    @BeforeEach
    void Setup() {
        Date date = new Date();
        book = new Book("췍", date, "하하");
        book2 = new Book("췍", date, "호호");
        book3 = new Book("췍", date, "헤헤");
    }

    @Test
    @DisplayName("제목과 출간일(예: 2024-01–01)이 같으면 같은 책으로 판단한다. 또한 List, Map, Set 등의 컬렉션에 넣어도 동일 객체로 판단한다.")
    void List_Set_Test() {
        assertEquals(book, book2, "두 객체는 같아야한다 equals 재정의");

        List<Book> BookArray = new ArrayList<>();
        BookArray.add(book);
        BookArray.remove(book2);

        assertEquals(0, BookArray.size(), "List에서 equals가 같으면 remove()가 정상 작동하여 크기가 0이어야 한다.");

        Set<Book> BookSet = new HashSet<>();
        BookSet.add(book);
        BookSet.add(book2);

        assertEquals(1, BookSet.size(), "Set은 중복추가가 안되기 때문에 크기가 1이여야한다.");

        book.SetTitle("다른 제목");
        boolean HasBook = BookSet.contains(book);
        assertFalse(HasBook, "타이틀이 바뀌면서 해시코드가 달라져서 기존에 넣은 책을 찾을 수 없음");
    }

    @Test
    @DisplayName("Collections.sort()를 사용하면 출간일 기준 오름차순(과거부터 최신순)으로 정렬된다.")
    void Book_Sort_Test() {
        book.SetTitle("빠른책");
        book2.SetTitle("중간책");
        book3.SetTitle("느린책");

        Date date = new Date();
        long now = date.getTime();
        book.SetDate(new Date(now - 1000));
        book2.SetDate(new Date(now));
        book3.SetDate(new Date(now + 1000));

        List<Book> BookList = new ArrayList<>();
        BookList.add(book3);
        BookList.add(book);
        BookList.add(book2);

        // 정렬 전 상태 검증 (첫 번째 책이 느린책인지 확인)
        assertEquals("느린책", BookList.get(0).GetTitle());

        Collections.sort(BookList);

        assertEquals("빠른책", BookList.get(0).GetTitle(), "가장 과거에 출간된 책이 맨 앞으로 와야 한다.");
        assertEquals("중간책", BookList.get(1).GetTitle(), "그다음 날짜의 책이 두 번째여야 한다.");
        assertEquals("느린책", BookList.get(2).GetTitle(), "가장 최근(미래)에 출간된 책이 마지막이어야 한다.");
    }

    @Test
    @DisplayName("clone()을 호출하면 원본과 데이터는 같지만 주소가 다른 깊은 복사 객체가 생성된다.")
    void Book_Clone_Test() {
        Book CloneBook = book.clone();
        assertEquals(book, CloneBook, "복사된 객체는 원본과 데이터 내용이 같아야 합니다.");
        assertNotSame(book, CloneBook, "내용은 같아도 메모리상의 주소는 서로 달라야 합니다.");

        CloneBook.SetTitle("췍이아닌 책");

        assertEquals("췍", book.GetTitle(), "사본을 수정해도 원본의 데이터는 보호되어야 합니다.");
        assertEquals("췍이아닌 책", CloneBook.GetTitle(), "사본의 데이터만 독립적으로 변경되어야 합니다.");
    }
}

``````

## 어려웠던 점

- 동등성 규칙을 재정의 하여 새로운 기준을 정의하는게 좀 어려웠다고 느꼈다.
