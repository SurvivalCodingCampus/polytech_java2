# 2026-06-23 제네릭, 열거형

## 오늘 배운 내용

- 타입이 없으면 런타임에러가 나기쉽고, IDE가 컴파일 에러를 찾을 수 없다.
- 제네릭을 쓰는 이유는 타입을 나중에 원하는 타입으로 바꿀수 있고, 타입안전효과 있어서
-

## 실습 코드

```java
public class Pocket<E> {
    private E data;

    public void put(E data) {
        this.data = data;
    }

    public E get() {
        return this.data;
    }
}
```

## 기억할것

- 타입을 원하는 형태로 정의할 수 있는 제너릭 자주 쓸것
- 제너릭을 사용해서 타입안전효과를 받을 것
- 타입에 제약을 추가할 때는 타입 옆에 extends 할 것
