# 2026-06-23 다형성

## 다형성

- 인터페이스나 메서드 호출이 객체의 타입에 따라 다르게 동작하는 성질
- 현재 폴더의 company를 기준으로
    - `Thing thing = new book("이름", (가격), "색", (무게), "ISBN"` 처럼 Thing을 통해 book을 관리할 수 있음.
- 다형성을 이용하면 ArrayList와 같은 방식으로 book 과 computer를 함께 관리 할 수 있음.
- 단, 컴파일러 에서는 `Thing thing` 을 보고있기 때문에 `book`에만 있는 `isbn`을 `getIsbn()`을 통해 볼 수 없다.

### 타입 변경

- `Thing` 아래에 `Book` 이 있다면, 앞에 `(Book) thing` 으로 캐스팅을 할 수 있다.
    - 단 실패하거나, 다른 객체일 수 있으므로 아래 코드를 응용하여 작성해야한다
      ```java
      if(thing instanceof Book) {
          Book book = (Book) thing
      }    
      ```

### 정리

- 서로 다른 자식 클래스의 인스턴스를 동일한 부모 클래스 타입으로 다룰 수 있음
    - 상속에 의한 is-a 관계가 성립될 경우, 인스턴스를 부모 클래스 타입 변수에 대입 가능
    - 실제 실행 메서드는 인스턴스의 실제 타입에 따라 달라질 수 있음
- 상자의 타입, 내용의 타입
    - 어떤 멤버를 이용할 수 있는가 - 상자의 타입
    - 멤버가 어떻게 움직이는가 - 내용의 타입
- 타입 변경
    - 캐스트 연산자를 이용하여 강제로 대입시킬 수 있음
    - 부정한 대입이 발생할 경우 ClassCastException이 발생함
        - `instanceof` 를 이용하여 캐치할 수 있음
- 다형성
    - 다른 인스턴스를 동일시하여, 부모 클래스 타입 배열에 담을 수 있음
    - 부모 클래스타입의 인수나 리턴 값을 이용하여 다른 클래스를 모아서 처리할 수 있음
    - 동일시 취급을 하지만, 각 인스턴스는 각 클래스의 정의를 따르고 동작함.

## 과제 중 알게된 점

- 다형성을 이용하여 일종의 필터링처럼 역할을 나눌 수 있다 라는 것을 알게됨
- 과제 내용을 만들 때, 이 기능이 중복되어 가져야하는 클래스들을 파악하는데 어려움을 느낌
- 인터페이스 - 인터페이스는 다중 상속이 가능하고, 인터페이스 - 추상클래스는 상속 할 수 없다는 것을 알게됨
